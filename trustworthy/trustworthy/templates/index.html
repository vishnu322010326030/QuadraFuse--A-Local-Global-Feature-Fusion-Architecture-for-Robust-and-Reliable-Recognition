Face Recognition with Occlusion Handling

An intelligent AI system that recognizes faces even when partially occluded by masks. Using quadrant-based feature extraction and attention mechanisms, this system adapts its recognition strategy based on which facial features are visible, ensuring reliable identification in real-world scenarios.

✨
Key Features & Innovation

🎯

Adaptive Recognition

Automatically detects occlusions (masks) and intelligently selects which facial quadrants to use for recognition, ensuring accuracy even with partial visibility.

🧩

Quadrant Fusion

Divides faces into 4 strategic quadrants (top-left, top-right, bottom-left, bottom-right) and fuses features from unoccluded regions for robust identification.

🔒

Trustworthy AI

Built with transparency in mind - the system explicitly shows which features are used for recognition, making the decision-making process interpretable and trustworthy.

Recognition Flow

1

Input Detection

Face image is analyzed to detect presence of occlusions (mask)

2

Quadrant Selection

• Mask → Use top 2 quadrants (eyes, forehead)
• Sunglasses → Use bottom 2 quadrants (nose, mouth)
• No occlusion → Use all 4 quadrants

3

Feature Fusion

Selected quadrants are processed through attention mechanism to extract and fuse relevant features

4

Identity Prediction

Final classification determines the person's identity using the extracted partial features

Results & Performance

🎉 Key Achievement

The model successfully recognizes faces with masks even though those masked samples were never part of training. This demonstrates the power of quadrant fusion and attention mechanisms that allow the model to rely on partial local features (forehead, eyes) when part of the face is occluded.

Key Insights

🎯

Mask Training Strategy

The mask class in training teaches the model to detect lower face occlusion, not to identify specific individuals with masks

🧠

Attention Mechanism

Successfully focuses on discriminative features in visible portions (eyes, forehead) when masks are detected

✨

Generalization

Model generalizes to unseen masked samples by leveraging partial local features from unoccluded quadrants

Technical Implementation

📁 Dataset Structure

Organized folder structure with separate directories for each identity (face1-5) and occlusion types (masked, sunglasses), enabling systematic training and evaluation.

🧠 Architecture

Deep learning model with quadrant extraction layers, attention mechanisms for feature weighting, and fusion layers for combining information from multiple regions.

🎯 Trustworthiness

Transparent decision-making with explainable quadrant selection, ensuring users understand which facial features contributed to the recognition decision.
